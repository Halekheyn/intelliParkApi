Bloque 3B — Login de usuarios con JWT

Ya tenemos registro. Ahora vamos a permitir que un usuario pueda iniciar sesión.

El endpoint será:

POST /api/auth/login
A. Objetivo del módulo

Crear el login del sistema IntelliPark para que el usuario pueda:

Enviar correo y contraseña.
Validar que el usuario exista.
Validar que esté activo.
Comparar la contraseña enviada contra la contraseña cifrada en base de datos.
Generar un token JWT.
Devolver los datos básicos del usuario y el token.

B. Archivos que vamos a crear o modificar
src/
  routes/
    auth.routes.js

  controllers/
    auth.controller.js

  services/
    auth.service.js

  utils/
    password.util.js
    jwt.util.js

C. Responsabilidad de cada archivo

| Archivo              | Responsabilidad                                            |
| -------------------- | ---------------------------------------------------------- |
| `auth.routes.js`     | Agrega la ruta `/login`.                                   |
| `auth.controller.js` | Recibe la petición HTTP del login.                         |
| `auth.service.js`    | Valida credenciales y coordina la lógica de autenticación. |
| `password.util.js`   | Agrega comparación de contraseñas con bcrypt.              |
| `jwt.util.js`        | Genera el token JWT.                                       |


E. Explicación del flujo

Cuando haces login:

POST /api/auth/login

el flujo es:

1. auth.routes.js recibe /login.
2. auth.controller.js toma email y password del body.
3. auth.service.js valida que lleguen los datos.
4. users.db.js busca el usuario por email.
5. password.util.js compara la contraseña.
6. jwt.util.js genera el token.
7. auth.controller.js responde con el token y los datos básicos del usuario.
